HTML Basics

HTML (Hypertext Markup Language) is the code used to structure a web page and it's content.For examplw, content could be structured within a set of paragraphs, a list of bulleted points, or using images and data tables. As the title suggests, this section will cover the basics of HTML and it's functions.

What is HTML?

HTML is a markup language that defines the structure of your content. HTML consists of a series of elements, which you can enclose, or wrap, different parts of the content to make it appear a certain way, or act a certain way. For example, look at the following lone of content

The Anatomy of an HTML element

The main parts of the HTML element are:

  1. The opening tag: This is the name of the element (in this case, p) wrapped in opening and closing angle brackets. This states where the element begins or starts to take effect - in this case where the paragraph begins

  2. The Closing Tag: This is the same as the opening tag, except that it includes a foward slash before the element name. This states where the element ends - in this case, where the paragraph ends. failing to add the closing tag is one of the beginner errors and can lead to stramge results.

  3. The Content: This is the content of the element. which in this case is just text.

  4. The Element: The opening tag and the closing tag coming together with the content to create the element.

Elements can also have attributes

Attributes contain extra information about the element that you don't want to appear in the actual content. Here 'class ' is the attribute name and 'editor-note' is the attrubute value.The class attribute allows you to give the element a non-unique identifier that can be used to target it (any other element with the same class value) with style information and other things

An attribute should always have the following


  1. A space between it and the element name (or the previous attribute, if the element has more than one attribiue).

  2. The attribute name followed by an equal sign.

  3. The attribute value wrapped in quotation marks.

That wraps up the basics of idividual HTML elements. Now we look at how individual elements behave when they are combined to form an entire HTML page.

Images

As it was mentioned earlier, the 'img' tag embeds an image into the page in the position that it appears. It does this via the 'src' attribite, which contains the path to the image file.

We incude an 'alt' attribute. In this attribute, you specify descriptive text for users who cannot see the image, possibly for the following reasons.

  1. They are visually impaired. Users with significant visual impairments often use tools called screen readers to read out the alt text to them
  2. Something has gone wrong causing the image not to display.

Links are very important, they are what makes the web a web. To add a link we need to use the 'a' element, being short for anchor.The following steps are to turn text in your paragraph into a link

  3. Fill the value of the attribute with thw web adress you want the link to link to.

The Anatomy of a CSS ruleset

The whole structure is called a ruleset. Note the names of the individual parts

  • Selector
    This is the HTML element name at the start of the ruleset. It defines the element(s) to be styled (in this example, 'p' element) to style a different element change the selector

  • Declaration
    This is a single rule like color: red;. its specifies which of the element's properties you want to style

  • Properties
    These are ways in whic you can style an HTML element.(in this example, colour is the property of the 'p' element). In CSS you choose which properties you want to affect in the rule.

  • Property Value
    To the right of the property, after the colon, is the property value. This chooses one out of many possible appearances for a given property.

Other important parts of the syntax:

  • Apart from the selector, each ruleset must be wrapped with curle braces {}
  • Within each declaration, you must use a colon to separate the property from it's value(s).
  • Within each ruleset, you must use a semicolon to separate each declaration from the next one

Different types of selectors

There are many different types of selectors. The examples above use element selectors which select all the elements of a given type. But we can also make more specific selectioms as well. Here are are some of the more common types of selectors

Selector Name What does it select? Example
Element Selector All HTML elements of the specified type P selects the paragraph elementt
ID selector The element on the page with a specified ID. On a given HTML page, each id value should be unique #my-id selects <p id="my-id"> or a <a id="my-id">
Class selector The elements(s) on the page within the page with a specified class. Multiole instances of the same class can appear on the same page .my-class selects <p class="my-class">and <a class="my-id">
Attribute Selector The element(s) on the page with a specific atribute. img[src] selects <img src="myimage.png"> but not <img>
Pseudo-class selector The specified element(s), but only when in the specified state (for example, when a cursor hovers over a link) a: hover selects <a>, but only when the mouse pointer is hovering over a link.
